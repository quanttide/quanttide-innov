# 步骤二：评估维度与权重

> 目标：确定评分维度及其权重，使决策可量化。

## 评分模型

采用**加权评分法**，总分 = Σ(维度得分 × 权重)。

### 默认评估维度

请根据步骤一的实际需求调整维度和权重（合计 100%）：

| 维度 | 默认权重 | 说明 |
|------|---------|------|
| 开发效率 | 20% | 开发速度、热重载体验、代码复用率 |
| 性能体验 | 20% | 启动速度、渲染性能、帧率稳定性 |
| 生态成熟度 | 15% | 三方库丰富度、社区活跃度、文档质量 |
| 团队适配成本 | 15% | 学习曲线、已有技能复用度、招聘难度 |
| 原生能力可达性 | 10% | 原生 API 调用便利性、自定义原生模块成本 |
| 可维护性 | 10% | 代码可读性、重构成本、架构灵活性 |
| 长期发展潜力 | 10% | 框架公司支持力度、版本稳定性、技术趋势 |

### 评分标准

每个维度采用 **1-5 分** 评分：

| 分值 | 含义 |
|------|------|
| 1 | 差（严重不满足需求） |
| 2 | 及格（基本功能可满足） |
| 3 | 良好（满足大部分需求） |
| 4 | 优秀（显著满足需求） |
| 5 | 卓越（超出预期） |
