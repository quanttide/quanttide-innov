# 步骤一：需求定义

> 目标：明确团队对跨平台框架的核心需求，作为后续评估的基准。

## 填写说明

请从以下维度梳理项目/团队的实际需求，越具体越好。

### 1. 项目特征

| 问题 | 回答 |
|------|------|
| 目标平台有哪些？ | iOS / Android / Web / 桌面 / 嵌入式 |
| 项目类型是什么？ | 全新项目 / 现有项目迁移 / 混合集成 |
| 预期的用户规模？ | 内部工具 / 小规模 / 中等 / 大规模 |
| 对性能的敏感度？ | 低 / 中 / 高（如动画、滑动列表、实时渲染） |
| 是否需要原生功能集成？ | 蓝牙 / 相机 / 传感器 / 支付 / 其他 |

### 2. 团队能力

| 问题 | 回答 |
|------|------|
| 团队当前擅长什么语言？ | Dart / JavaScript/TypeScript / 两者皆可 |
| 是否有移动端开发经验？ | 原生 iOS / 原生 Android / Flutter / RN / 无 |
| 团队规模？ | 1 人 / 2-5 人 / 5-10 人 / 10+ |
| 招聘难度预期？ | 易 / 中 / 难（本地 Flutter / RN 开发者供给如何） |

### 3. 业务约束

| 问题 | 回答 |
|------|------|
| 上线时间要求？ | 紧急（3个月以内） / 正常（3-6个月） / 充裕 |
| 长期维护周期？ | 1年以内 / 1-3年 / 3年以上 |
| 是否需要热更新 / 动态化？ | 是 / 否 / 未来可能需要 |
| 是否有现有技术栈绑定？ | 如：已用 React Web、Dart 后端等 |

### 4. 生态与三方依赖

| 问题 | 回答 |
|------|------|
| 项目需要哪些关键三方库？ | 地图 / 图表 / 推送 / 支付 / 音视频 |
| 是否需要与现有系统集成？ | API / SDK / 原生模块 |
